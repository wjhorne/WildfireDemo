# AI Instructions for PipelineSolution

This file guides AI assistants (Claude, Gemini, Codex, pi, …) on how to use the
wildfire strategy-optimization pipeline. The pipeline is driven through its CLI
(`pipeline.py` / `query.sh`); no MCP server is required.

## Goal

Given an initial terrain (or seed), a fire start, and firefighter starting
locations, recommend **where the firefighters should go** to minimize burned
area, respecting their motion restrictions (terrain-dependent speed, water
impassability, the 2-turn mountain cost). An **agentic workflow** achieves this:
the agent parses the request, drives the pipeline to search strategies,
interprets the ranking, and (optionally) refines.

## Setup (once)

```bash
./setup.sh
```

## How to invoke

```bash
./query.sh "<plain-language request>" [--format text|pdf] [--output FILE] [--validate]
# or
python pipeline.py --query "..." --format pdf --output report.pdf
```

## What you can ask for

### Strategy optimization (default)
- "Best strategy with 3 firefighters to minimize burned area"
- "64x64 grid, 180 turns, seed 12345, 3 firefighters"
- "fire at 30,30, firefighters start at 5,5 and 25,25, 2 firefighters"

### Reports
- "Generate a PDF report with 3 firefighters"  → use `--format pdf --output report.pdf`

### Validation
- "Validate against the baseline" / "example configuration" → use `--validate`

## Agentic workflow to follow

When a user asks something like *"Given this terrain and 3 firefighters starting
in the southwest, what's the best strategy?"*:

1. **Parse** the request into pipeline parameters (grid, turns, seed, firefighters,
   fire/firefighter starts). Ask the user for any missing required detail
   (firefighter count/starts, fire start, or a seed/terrain).
2. **Run** the optimizer:
   `./query.sh "<query>"` (text) to get the ranked strategy comparison and the
   recommended strategy.
3. **Interpret** the output: report the **recommended strategy**, its final burned
   area vs. the alternatives, and the fire peak.
4. **Produce a deliverable** if asked: `--format pdf --output report.pdf`.
5. **Refine** if useful: adjust parameters (more firefighters, different starts,
   `p_extinguish`) and re-run to compare; or, if implementing a novelty, extend
   `strategy_optimizer.py` with a richer search and re-run.

## Output you will see

- **Text**: a ranked `STRATEGY COMPARISON` table (burned / fuel / depleted / peak)
  and a `RECOMMENDED STRATEGY` block with final burned area and fuel remaining.
- **PDF** (3 pages): summary + recommended strategy, burned-area-over-time per
  candidate, and the final landscape under the recommended strategy.

## Determinism & validation

- All runs are deterministic for a fixed seed; the chosen strategy is part of the
  fixed input.
- `--validate` confirms the pipeline calls the same engine as the hand-run
  baseline (`DevelopmentSolution/example_run/wildfire_counts.csv`), exact match.

## Where to add a novelty

`strategy_optimizer.py` is the extension point: add candidate strategies (e.g.,
beam search over waypoints, ensemble across seeds, goal-seeking on firefighter
count, or a multi-objective Pareto over burned area vs. travel effort). The
engine's pluggable seam is `run_simulation(config, strategy_fn=...)`.

## Troubleshooting

- Run `./setup.sh` first.
- Ensure you are in the `PipelineSolution/` directory and `.venv` exists.
- Dependencies: `numpy==1.26.4`, `matplotlib==3.8.4` (Python ≥3.9).
